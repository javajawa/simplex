Simplex Solver — Some Help

History & About

The simplex solver was originally written whilst revising for an Operations Research exam. The past papers were somewhat useless in the respect that there were no sample answers, and the dependence of each part of a question on the previous one caused problems.

When you add to this the number of mistakes that creep in on the third day of adding up column and subtracting rows, it didn't take long for disagreements to form between members of the revision group. I took the decision that coding something which could perform a pivot operation would be useful.

That should have been it done forever — except that I happened to end up writing a rational number handling library one evening (as one does). So, the project got resurrect (and, in fact, entirely rewritten) as a powerful use case for that extremely small library.

You can read more on my blog.

What does this tool do?

The tool has three main features:

  • It will take a set of variables and constraints, and draw out a simplex tableau that describes them.
  • It will perform a pivot operation, replacing a basic variable with a non-basic variable, and updating all of the cells in the tableau accordingly.
  • Keeps a history of all the tableau's seen

How do I enter values?

Although you can't just copy-paste equations into a box (yet), I tried to make the number handling usefully clever. It will accept integers (eg, '2', '-3'), decimal strings '2.22', '-1.5', and rational numbers '8/3' , '-74/6'; number are then all stored as cancelled down rational numbers

Auto Solve

Be careful of the auto-solve. There will always be the temptation to do some piece of coursework by using the auto solve, and then reformatting the output. However, you're still going to have to do some work, because the rationals this thing throws out will give away the fact a computer it doing in the work — in reality, you don't want to always cancel things down; having everything over the same common denominator is generally more useful.

Furthermore, the auto solver may not be entirely perfect.

Why is there a greyed-out dropped-down saying 'max'?

When doing a simplex problem, you can either be asked to minimise or maximise the main equation (the objective row). The procedure is almost identical; however, I finished this code mainly for the sake of being able to take it off the to-do list. Therefore, the minor changes required to make the auto solver solve minimisation problems wasn't implemented. You're welcome to patch the code, though!

What is x0 and the top row?

x0 is a value that represents your objective function. If you were maximising 3x1 + 4x2, then we say that x0 = 3x1 + 4x2, or x0 - 3x1 + 4x2 = 0.
